# 🚀 CafeManager — Система автоматизации заведений общественного питания

## 📖 Общее описание проекта
**CafeManager** — это высокопроизводительная информационная система, спроектированная для цифровой трансформации бизнес-процессов ООО «Кофе-Хауз». Проект переводит управление заведением с ручного бумажного учета на полностью автоматизированный цифровой контур, обеспечивая прозрачность транзакций, контроль себестоимости в реальном времени и повышение качества обслуживания гостей.

## 🎯 Цели и задачи
* **Цель:** Проектирование и программная реализация прототипа информационной системы для автоматизации полного цикла обслуживания.
* **Ключевые задачи:**
    * Анализ предметной области и формирование функциональных требований.
    * Разработка нормализованной реляционной модели данных с обеспечением ссылочной целостности.
    * Реализация ролевой модели доступа для администраторов, официантов, поваров и бариста.
    * Обеспечение высокой конкурентности транзакций в условиях пиковых нагрузок.

## 🛠 Технологический стек
* **Серверная часть:** .NET 8, Web API, Entity Framework Core.
* **Клиентская часть:** C# WPF, архитектура MVVM (Model-View-ViewModel) для разделения логики и представления.
* **Система управления базами данных:** MS SQL Server 2022.
* **Инструменты проектирования:** SQL Server Management Studio, UML (диаграммы прецедентов, диаграммы последовательности).

## 🗄 Структура базы данных
База данных построена на принципе строгой типизации и обеспечения целостности данных:
* **Справочники:** Категории блюд, актуальный прайс-лист, реестр сотрудников.
* **Транзакционный блок:** Система управления столиками и заказами.
* **История и отчетность:** Таблицы позиций заказов, история смен персонала, логирование финансовых транзакций.
* **Оптимизация:** Применение индексов (покрывающих и составных), хранимых процедур для бизнес-логики и представлений для аналитики.

## ⚙️ Функциональные возможности
* **KDS (Kitchen Display System):** Мгновенная визуализация очереди заказов для кухни.
* **Управление доступом:** Гибкая система прав, разграничивающая возможности персонала.
* **Аналитический модуль:** Агрегированные отчеты по выручке, популярности позиций и эффективности работы персонала.

## 🧪 Тестирование и качество
Система прошла многоуровневое тестирование:
1. **Unit-тестирование (xUnit):** Покрытие критических бизнес-логик (расчет себестоимости, валидация DTO) на 78%.
2. **Интеграционное тестирование:** Эмуляция взаимодействия API с СУБД для проверки транзакций.
3. **Нагрузочное тестирование:** Эмуляция более 50 одновременных сессий, подтвердившая стабильность отклика (среднее время ответа API — 18-45 мс).
